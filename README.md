# Restaurant Table Occupancy Tracker

Система автоматизированной аналитики занятости ресторанных столиков на основе компьютерного зрения. Проект предназначен для сбора статистики об оборачиваемости посадочных мест, что позволяет менеджменту оптимизировать рассадку и оценивать реальную нагрузку на зал.

## Основной функционал

* **Детекция объектов в реальном времени:** Использование моделей YOLOv8 (Ultralytics) для обнаружения посетителей.
* **Интерактивный выбор зоны (ROI):** Возможность вручную выделить целевой столик на видео при запуске, что обеспечивает гибкость без изменения кода под разные ракурсы камер.
* **Интеллектуальный трекинг статуса:** Система фильтрации кратковременных сбоев детекции (морганий) через настраиваемый буфер кадров.
* **Аналитика оборачиваемости:** Автоматический расчет среднего времени нахождения клиента за столом на основе лога событий (приход/уход).
* **Визуализация:** Генерация итогового видеофайла с наложением рамок объектов и цветовой индикацией статуса стола.

## Технологический стек

* **Python 3.10+**
* **OpenCV:** Обработка видеопотока, работа с графическими примитивами и интерфейс выбора зоны интереса.
* **Ultralytics (YOLOv8):** Инференс нейросетевой модели детекции.
* **Pandas:** Обработка логов событий и формирование статистических отчетов.
* **Pydantic / Pydantic-settings:** Валидация конфигурации и управление гиперпараметрами проекта.

## Архитектура проекта

Проект реализован по модульному принципу, что упрощает поддержку и тестирование отдельных компонентов:

* `main.py` — управляющий скрипт, координирующий работу видеопотока, детектора и трекера.
* `core/detector.py` — обертка над нейросетью для извлечения координат объектов.
* `core/tracker.py` — логика определения состояния стола и ведение истории событий.
* `core/config.py` — централизованное управление параметрами системы (пороги уверенности, пути к моделям, настройки визуализации).
* `core/analytics.py` — модуль для расчета бизнес-метрик на основе накопленных данных.

## Алгоритмические особенности

В ходе разработки были решены ключевые проблемы классической 2D-детекции:

1.  **Ложные срабатывания от прохожих:** Для исключения срабатываний от людей, проходящих между камерой и столом, внедрена проверка по нижней центральной точке (footprint). Стол считается занятым только в том случае, если точка "ног" человека находится внутри выделенной зоны стола.
2.  **Проблема "холодного старта":** Реализован алгоритм начальной инициализации. В течение первых 15 кадров система накапливает данные, чтобы корректно определить статус стола (свободен или занят) в момент начала записи, не создавая ложных событий о "приходе" клиента, если он уже сидел за столом.
3.  **Оптимизация окна предпросмотра:** Добавлена поддержка масштабирования окна выбора зоны под разрешение монитора, что позволяет комфортно работать с видео высокого разрешения (4K/FullHD).

## Установка и запуск

1.  Клонируйте репозиторий и перейдите в папку проекта.
2.  Создайте и активируйте виртуальное окружение:
    ```bash
    python -m venv .venv
    # Для Windows:
    .\venv\Scripts\activate
    # Для Linux/macOS:
    source .venv/bin/activate
    ```
3.  Установите необходимые зависимости:
    ```bash
    pip install -r requirements.txt
    ```
4.  Запустите обработку видео:
    ```bash
    python main.py --video "путь_к_файлу.mp4"
    ```

## Конфигурация

Все ключевые настройки проекта доступны в файле `core/config.py`:
* `CONFIDENCE` — порог уверенности детектора (по умолчанию 0.35).
* `BUFFER_FRAMES` — количество кадров для подтверждения смены статуса стола.
* `YOLO_MODEL_PATH` — выбор модели (yolov8n.pt, yolov8s.pt, yolov8m.pt) в зависимости от требуемой точности и мощности железа.
